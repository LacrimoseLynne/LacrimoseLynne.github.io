Hexo框架学习笔记

Hexo常用指令

npm install hexo -g ————– 安装Hexo

npm update hexo -g ————– 升级

hexo init 文件名 ————– 初始化博客 文件名指定了博客存放的目录

hexo n(new) “文章名” ———– 新建一篇文章

hexo g(generate) ———- 生成静态网页

hexo s(server) ———– 可以在本地预览网站效果(Hexo自带浏览器 地址: https://localhost:4000)

hexo d(deploy) ———– 部署

hexo server #Hexo 会监视文件变动并自动更新，无须重启服务器

hexo server -s ————- 静态模式

hexo server -p 5000 ———– 更改端口

hexo server -i 192.168.1.1 ———— 自定义 IP

hexo clean ————- 清除缓存，若是网页正常情况下可以忽略这条命令

推送网站

根目录下的_config.yml文件为站点配置文件

参数 描述
title 网站标题
subtitle 网站副标题
description 网站描述
author 作者名字
language 网站使用的语言
timezone 网站所处的时区 默认使用计算机的时区

更换主题

根目录里面的themes文件夹中的_config.yml文件为主题配置文件

官方提供了很多个性化主题 https://hexo.io/themes/ 可供下载安装

图片下面有链接可以进入到Github里获取源码

复制仓库地址

进入myblog项目的根目录下 执行以下命令:

git clone 仓库地址 themes/主题名

下载完毕后 修改_config.xml文件 更换新主题:

themes : 主题名

随后执行

hexo clean

hexo generate

重新发布静态网站 也可以使用hexo server在本地查看效果

关联Hexo与Github

打开站点的配置文件_config.yml，翻到最后修改为：

deploy:

type: git

repo: 这里填入你之前在GitHub上创建仓库的完整路径，记得加上 .git

branch: master

保存站点配置文件

其实就是给hexo d 这个命令做相应的配置，让hexo知道你要把blog部署在哪个位置，很显然，我们部署在我们GitHub的仓库里。最后安装Git部署插件，输入命令：

npm install hexo-deployer-git –save

这时，我们分别输入三条命令：

hexo clean

hexo g

hexo d

完成后，打开浏览器，在地址栏输入你的放置个人网站的仓库路径，即 http://xxxx.github.io

至此你的博客已经上线了，可以在网络上被访问了
